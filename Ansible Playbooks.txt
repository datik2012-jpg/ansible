////////////////Ansible when conditions ///////////

⭐ 1. Condition based on a variable value
yaml
- name: Install nginx only if enabled
  yum:
    name: nginx
    state: present
  when: nginx_enabled == true
Use this when a boolean or flag controls a task.

⭐ 2. Condition based on string comparison
yaml
- name: Restart service only on production
  service:
    name: myapp
    state: restarted
  when: env == "prod"
⭐ 3. Condition based on OS family
yaml
- name: Install package on RedHat systems
  yum:
    name: httpd
    state: present
  when: ansible_facts.os_family == "RedHat"
⭐ 4. Condition based on multiple conditions (AND)
yaml
- name: Install nginx only on prod AND RedHat
  yum:
    name: nginx
    state: present
  when:
    - env == "prod"
    - ansible_facts.os_family == "RedHat"
⭐ 5. OR condition
yaml
- name: Run task on prod OR staging
  debug:
    msg: "Environment is prod or staging"
  when: env in ["prod", "staging"]
⭐ 6. Condition based on file existence
yaml
- name: Run only if config file exists
  debug:
    msg: "Config found"
  when: ansible_facts['files']['/etc/myapp.conf'].exists
⭐ 7. Condition based on command result
yaml
- name: Check if user exists
  command: id dani
  register: user_check
  ignore_errors: yes

- name: Create user if missing
  user:
    name: dani
  when: user_check.rc != 0
⭐ 8. Condition inside a loop
yaml
- name: Install only selected packages
  yum:
    name: "{{ item.name }}"
    state: present
  loop: "{{ packages }}"
  when: item.enabled == true
⭐ 9. Condition based on host group
yaml
- name: Run only on web servers
  debug:
    msg: "This is a web server"
  when: "'web' in group_names"
⭐ 10. Condition based on fact (RAM, CPU, etc.)
yaml
- name: Run only if RAM > 4GB
  debug:
    msg: "Enough memory"
  when: ansible_facts.memtotal_mb > 4096
