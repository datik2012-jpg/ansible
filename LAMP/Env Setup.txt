Note:

These are system packages being installed on your target servers (CentOS/RHEL-based systems) to ensure that Ansible can manage security and firewall settings properly:

python3-libselinux: This provides Python bindings for SELinux (Security-Enhanced Linux). Ansible needs this package on the target server to manage file contexts, permissions, and security policies. Without it, any Ansible task attempting to copy files or change permissions with SELinux enabled would fail.

python3-libsemanage: This provides the Python bindings for configuring and managing SELinux policies dynamically (like changing SELinux booleans or port contexts).

firewalld: This is the default dynamic firewall management tool for modern Linux distributions. Installing it ensures the firewall service is available so that a later task in your playbook can safely open up ports (like port 80 for your web server and 3306 for your database).

golden rule:


1. Read the Ansible Module Documentation (The Proactive Way)
Ansible documents these requirements explicitly. If you look up the documentation for the copy, template, file, or seboolean modules, there is a Requirements section right at the top.

For example, the documentation for any module modifying files on a RedHat/CentOS system states:

libselinux-python (for Python 2) or python3-libselinux (for Python 3) is required on the target host if SELinux is enabled.

Before using a module you aren't 100% familiar with, a quick glance at its Ansible doc page will tell you what dependencies the target machine needs.

2. The Golden Rule of Ansible: Know Your OS
Ansible doesn't have magic built-in powers; it just executes Python scripts on the target server.

RedHat, CentOS, and Rocky Linux have SELinux enabled by default.

Because SELinux protects the file system, any tool trying to modify files (like Ansible) must interact with the SELinux API.

Since Ansible is written in Python, it needs the Python-to-SELinux bridge (python3-libselinux).

Once you know that a target server runs CentOS/RHEL and has SELinux active, you automatically know you need these packages.

3. The "Fail Fast" Approach (The Reactive Way)In reality, even seasoned DevOps engineers don't memorize every package. They use a sandbox environment:They write a simple playbook.They run it against a test server.Ansible crashes with a very clear error:"msg": "SELinux coversions to public keys failed. Please install python3-libselinux."They add the package installation task to the top of the playbook.Summary Checklist for Future PlaybooksWhenever you target RHEL/CentOS systems, always ask yourself:Am I managing files, users, or permissions? $\rightarrow$ Install python3-libselinux.Am I managing firewalls? $\rightarrow$ Install firewalld and python3-libsemanage.Am I managing Apache/NGINX ports? $\rightarrow$ Install python3-libsemanage (needed to map ports in SELinux).
