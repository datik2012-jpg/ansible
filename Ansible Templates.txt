Ansible template example

/etc/ansible/hosts
ini
[web_servers]
web1 ansible_host=172.20.1.100
web2 ansible_host=172.20.1.101
web3 ansible_host=172.20.1.102
⚙️ playbook.yml
yaml
- hosts: web_servers
  tasks:
    - name: Copy index.html to remote servers
      template:
        src: index.html.j2
        dest: /var/www/nginx-default/index.html
🧠 index.html.j2
html
<!DOCTYPE html>
<html>
<body>
This is {{ inventory_hostname }} Server
</body>
</html>

Explanation:

The inventory file defines three web servers.

The playbook uses the template module to copy a Jinja2 file to each server.

The template dynamically inserts each host’s name using {{ inventory_hostname }}.

Each server will end up with a unique HTML file showing its own hostname.




Ansible template example

/etc/resolv.conf.j2
jinja2
{% for name_server in name_servers %}
nameserver      {{ name_server }}
{% endfor %}
⚙️ Variable definition
yaml
name_servers:
  - 10.1.1.2
  - 10.1.1.3
  - 8.8.8.8
📄 Rendered output (/etc/resolv.conf)
bash
nameserver 10.1.1.2
nameserver 10.1.1.3
nameserver 8.8.8.8
💡 Explanation:

The variable name_servers defines a list of DNS IPs.

The Jinja2 template loops through each item in that list.

When rendered, it produces a complete /etc/resolv.conf file with all nameservers listed.
